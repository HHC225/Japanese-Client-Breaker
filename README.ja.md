# Japanese-Hanko-Breaker

[English](README.md) | [简体中文](README.zh-CN.md) | [繁體中文](README.zh-TW.md) | [日本語](README.ja.md) | [한국어](README.ko.md)

`Japanese-Hanko-Breaker` は、日本クライアント向け防御に特化した harness であり、成果物を厳しい日本クライアント向けの説得・防御論点へ変換します。

このプロジェクトは、日本の銀行や保険会社を実際に説得する仕事の現場で使われました。文書を出して終わりではなく、レビューを切り抜け、間接的な異議に対応し、相手が面子を保ったまま承認できる理由を作る必要がある場面のために作られています。

ステータス：現在も継続的に構築中です。

## これは何か

`Japanese-Hanko-Breaker` は、日本クライアント防御のために設計された、ひとつの目的に特化した実務向け harness です。

成果物を、厳しい日本企業クライアントのレビューに耐えられる説得・防御戦略へ変換します。

想定しているのは、内容の良し悪しだけでなく、次のような観点でも厳しく見られるレビュー文化です。

- MECE の厳密さ
- 網羅性
- 根拠の強さ
- 前例への感度
- 体裁と形式の整合性
- リスク意識
- 相手の面子を保てるコミュニケーション

これは汎用的な prompt 集ではありません。成果物を分析し、批判を予測し、多段の反論ツリーを作り、品質を検証し、最後に実務で使えるレポートにまとめるための構造化パイプラインです。

## なぜ作るのか

日本の大企業レビュー、特に銀行や保険の現場では、成果物そのものが良くても失敗することがあります。たとえば次のような場合です。

- ロジックに小さな飛躍がある
- 分類が MECE になっていない
- 根拠や理由の明示が足りない
- クライアントが社内で説明しにくい
- 承認することでレビュー担当者自身のリスクが高まると感じられる

この harness は、その現実に備えるためのものです。

## クイックスタート

このリポジトリは、Claude Code の中で使う前提で設計されています。agent を手作業で配線する必要はありません。

1. リポジトリを clone して移動します。

```bash
git clone https://github.com/HHC225/Japanese-Hanko-Breaker.git
cd Japanese-Hanko-Breaker
```

2. 成果物を `input/` に入れます。

対応する入力は、Excel、PDF、PowerPoint、Word、CSV、画像、プレーンテキストです。ひとつの成果物が複数ファイルにまたがる場合は、すべて `input/` に入れてください。

3. リポジトリのルートで Claude Code を起動します。

ローカル環境で標準の CLI エントリーポイントを使っている場合、通常は次のように起動します。

```bash
claude
```

4. 次のプロンプトを貼り付けます。

```text
`input/` に入っている成果物を厳しい日本クライアントの視点で分析し、フルの防御パイプラインを実行して HTML レポートを生成してください。
```

5. 生成されたレポート `_workspace/defense-report.html` を開きます。

## 必要要件

- Python 3.10+
- 前処理スクリプト用に `uv` が必要です。初回実行時に自動インストールされることがありますが、失敗した場合は手動でインストールしてください。

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

- 初回の依存解決と HTML レポート内の Google Fonts の読み込みにはインターネット接続が必要です。オフライン時はシステムフォントにフォールバックします。

## どう動くか

現在の設計は、5つの中核 agent で構成されています。

1. `deliverable-analyst`
   成果物を批判可能な粒度まで分解し、前提・依存関係・構造上の抜けを整理します。
2. `jp-client-critic`
   厳しい日本クライアントを模擬し、現実的な指摘や反論を生成します。
3. `persuasion-strategist`
   日本の商習慣に合わせた表現で、多段の反論ツリーを構築します。
4. `consulting-qa`
   McKinsey、BCG、Bain、Deloitte、Accenture の観点で論点をストレステストします。
5. `report-generator`
   全体を、会議準備に使える HTML レポートへまとめます。

## リポジトリ構成

- `input/`
  分析したい成果物ファイルをここに入れます。
- `_workspace/`
  実行時に生成されます。中間成果物と最終 HTML レポートが入ります。
- `CLAUDE.md`
  Claude Code 上でこのリポジトリを使うための実行指示です。
- `.claude/agents/`
  エンドツーエンドの防御パイプラインを構成する agent 定義。
- `.claude/skills/`
  再利用可能な skill、参照資料、オーケストレーションロジック。
- `japanese-client-characteristics-research.md`
  日本の銀行・金融クライアントの行動特性に関する調査メモ。

## 向いている用途

このプロジェクトは特に次のような用途に向いています。

- 厳しいクライアントレビューに備えるコンサルティングチーム
- 日本の銀行、保険会社、大企業に向き合うデリバリーチーム
- 提案を書くことよりも、提案を守り切ることが重要な戦略・PM チーム
- 高摩擦なエンタープライズ説得ワークフローを agent で組みたい運用者
