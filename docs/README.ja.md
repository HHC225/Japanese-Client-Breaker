# Japanese-Client-Breaker

[English](../README.md) | [简体中文](README.zh-CN.md) | [繁體中文](README.zh-TW.md) | [日本語](README.ja.md) | [한국어](README.ko.md)

`Japanese-Client-Breaker` は、承認リスク、間接的な異議、社内説明責任が重い、高緊張の日本企業レビュー向けに設計された防御 harness です。

このプロジェクトは、日本の銀行や保険会社を実際に説得する仕事を含む現場で使われました。文書を出して終わりではなく、高緊張のレビューを切り抜け、間接的な異議に対応し、関係者が社内で説明できる形で、面子を保ったまま承認理由を作る必要がある場面のために作られています。

ステータス：現在も継続的に構築中です。

## これは何か

`Japanese-Client-Breaker` は、ひとつの目的に特化した実務向けの防御 harness です。

成果物を、高緊張の日本企業レビューに耐えられる説得・防御戦略へ変換します。

想定しているのは、内容の良し悪しだけでなく、次のような観点でも厳しく見られるレビュー文化です。

- MECE の厳密さ
- 網羅性
- 根拠の強さ
- 前例への感度
- 体裁と形式の整合性
- リスク意識
- 相手の面子を保てるコミュニケーション

これは汎用的な prompt 集ではありません。成果物を分析し、批判を予測し、多段の反論ツリーを作り、品質を検証し、最後に実務で使えるレポートにまとめるための構造化パイプラインです。

## なぜ作るのか

高緊張の日本企業レビューでは、成果物そのものが良くても失敗することがあります。たとえば次のような場合です。

- ロジックに小さな飛躍がある
- 分類が MECE になっていない
- 根拠や理由の明示が足りない
- レビュー担当者が社内で正当化しにくい
- 承認することでレビュー担当者自身のリスクが高まると感じられる

この harness は、その現実に備えるためのものです。

## クイックスタート

このリポジトリは、Claude Code の中で使う前提で設計されています。agent を手作業で配線する必要はありません。

1. リポジトリを clone して移動します。

```bash
git clone https://github.com/HHC225/Japanese-Client-Breaker.git
cd Japanese-Client-Breaker
```

2. 成果物を `input/` に入れます。

対応する入力は、Excel、PDF、PowerPoint、Word、CSV、画像、プレーンテキストです。ひとつの成果物が複数ファイルにまたがる場合は、すべて `input/` に入れてください。

3. リポジトリのルートで Claude Code を起動します。

ローカル環境で標準の CLI エントリーポイントを使っている場合、通常は次のように起動します。

```bash
claude
```

4. 次のプロンプトを貼り付けます。

```text
`input/` に入っている成果物を、高緊張の日本企業レビュー案件として分析し、フルの防御パイプラインを実行して HTML レポートを生成してください。
```

5. 生成されたレポート `_workspace/latest/defense-report.html` を開きます（または `_workspace/run_YYYYMMDD_HHMMSS/`）。

## 必要要件

- Python 3.10+
- 前処理スクリプト用に `uv` が必要です。初回実行時に自動インストールされることがありますが、失敗した場合は手動でインストールしてください。

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

- 初回の依存解決と HTML レポート内の Google Fonts の読み込みにはインターネット接続が必要です。オフライン時はシステムフォントにフォールバックします。

## どう動くか

現在の設計は、5つの中核 agent で構成されています。

1. `deliverable-analyst`
   成果物を批判可能な粒度まで分解し、前提・依存関係・構造上の抜けを整理します。
2. `jp-client-critic`
   高緊張の日本企業レビュー担当者を模擬し、現実的な指摘や反論を生成します。
3. `persuasion-strategist`
   日本の商習慣に合わせた表現で、多段の反論ツリーを構築します。
4. `consulting-qa`
   McKinsey、BCG、Bain、Deloitte、Accenture の観点で論点をストレステストします。
5. `report-generator`
   全体を、会議準備に使える HTML レポートへまとめます。

## リポジトリ構成

- `input/`
  分析したい成果物ファイルをここに入れます。
- `_workspace/`
  実行時に生成されます。実行ごとにタイムスタンプ付きサブディレクトリ（`run_YYYYMMDD_HHMMSS/`）が作成され、`latest` シンボリックリンクが最新の実行を指します。
- `CLAUDE.md`
  Claude Code 上でこのリポジトリを使うための実行指示です。
- `.claude/agents/`
  エンドツーエンドの防御パイプラインを構成する agent 定義。
- `.claude/skills/`
  再利用可能な skill、参照資料、オーケストレーションロジック。
- `japanese-client-characteristics-research.md`
  日本の銀行・金融クライアントの行動特性に関する調査メモ。

## 向いている用途

このプロジェクトは特に次のような用途に向いています。

- 高緊張の日本企業レビューに備えるチーム
- 日本企業の承認圧力の下で動くデリバリーチーム
- 社内で説明・正当化できる提案が必要な戦略・PM チーム
- 異議対応と承認支援のワークフローを agent で組みたい運用者
